============================================================
APP IMPLEMENTATION (CodeIgniter) — quick map
============================================================

This project registers the design paths in `Vendor/app/Config/DesignRoutes.php` (and short aliases in `Vendor/app/Config/ManageRoutes.php`) and dispatches `DashBoard/UCP.app`, `MCP.app`, and `ACP.app` through `App\Controllers\DashBoard\Gateway` (PHP class lives under a `DashBoard` folder so the URL segment spelling matches without colliding with `App\Controllers\Dashboard` on case-insensitive disks). Short path aliases (`d`, `u`, `m`, `b`, …) remain valid.

Query string: **`M=`** carries the board route segment (e.g. `Settings`, `User/ProFile`); legacy **`Route=`** is still accepted by the app.

Implemented:
- **Forward** (existing controllers): UCP `User/ProFile` + `View` → profile; MCP/ACP `Content` + `Pending|List` → page sections list; ACP `Content` + `Add|Edit` → editor; ACP `Settings` + `General` → site settings form.
- **HTTP redirect**: UCP `Logout` → public home (flash) so manage mode ends.
- **Design map pages** (`site/design_map_page.php`, copy from `Config/DesignMapPages.php`): UCP auth + account links (`Login`, `Register`, `ForgotPassWord`, `ResetPassWord`, `VerifyEmail`, `Settings`, `Security`, `Notifications`, `Billing`, `PassWord`, `E-Mail`, `UserName`); MCP `Index`/`View` (moderator hub), `Content`/`Approve|Reject`, `Reports`, `Users`, `Logs`, `Appeals`; ACP `Index`/`View` (admin hub), `Settings`/`Security|Email|API`, `Users`, `Roles`, `Modules`, `Logs`, `Backups`, `Analytics`. Unknown `M`/`I` still resolve to a generic map page with the same layout.

Helpers in `Vendor/app/Helpers/site_routes_helper.php`: `site_design_url_information_index`, `site_design_url_user_dashboard`, `site_design_url_ucp`, `site_design_url_mcp`, `site_design_url_acp`, `site_design_request_route`.

The public SEO slug cannot equal the lowercase design board segment (`dashboard` by default); see `ManageRoutes::reservedPublicSlugSegments()`.
